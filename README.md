# 정규화
> 이상현상(대부분 중복)을 없애는 과정, 프로세스   
- 무결성, 단일성...
- 이로인해 정규화가 거듭될수록 DB에 중복된 데이터를 안넣기때문에 `원자성`, `용량`이 줄어듬

**단점**
- 귀찮다. 쿼리 많아진다. 복잡한 연산이 많아진다.
- => 속도가 느려진다. (항상은 아님)   
  
\* **반정규화:** 너무 느려질경우 반대로 정규화

### 제 1 정규화
- 하나의 셀에는 하나의 데이터만

### 제 2 정규화
- 부분 종속을 없애자

### 제 3 정규화
- 종속을 없애자

**`결국엔 1, 2, 3을 나눠서 외우려 하지말고 종속을 없애면 된다.`, `실무에서는 3 정규화면 충분하다.` + BCNF 정도!**

 **참고**
1. 테이블명은 단수, API 복수형
2. 후보키 고려 X
3. PK(2개 이상 컬럼 조합) X
